Intro:

CSS Grid Layout is a two dimensional grid system.

With the implementation of CSS Grid in browsers, CSS Grid has been evangelised as being the most progressive standard in the implementation of web layouts. CSS Grid has been coined as the method that can completely solve the common issues that arise when laying out web-based elements.

I wanted to research and develop ways that we could use this in the design team at JH and would like to share how we push the boundaries of creativity, coming up with innovative layouts for our clients’ eCommerce sites.

Css Grid eCommerce Adaptation

What is CSS Grid?

CSS Grid Layout is a two dimensional grid system, it involves intersecting of a set of horizontal and vertical lines, one set defining columns and the other the rows. The key features of using grids that sets it apart from other frameworks and CSS layout standards like floats and flexbox are:

Fixed track:

Fixed track sizes using pixels

Flexible sizes:

Flexible sizes with percentage

[fr unit]

With the new [fr unit] this will automatically calculate space in a container.

Any combination

Any combination of some or all of these features will give you fine grain control over the layout of elements on a grid.

How does Grid works?

The grid container is created by declaring display: grid; or display: inline-grid; all the direct children of the element will become an grid item.

The Editorial Section

The Editorial Section has two main columns, that have been given the equal amount of space, this was done by by using grid-template-columns: 1fr 1fr.

Fractional unit:

1fr represents one "fractional unit", so to use 1fr 1fr or repeat(2, 1fr) this means the entire column width is divided into two. Each individual column will be given the exact same amount of width size.

Grid-gap

Grid-gap automatically calculates and takes away from the overall width and height of the column. This will give you gap space between the columns and rows, I have set this to 10px vertical and 10px horizontal.

Catalogue Section

The first column of the catalogue section is divided into two separate divs and the benefit of using grids opposed to something like flex or floats is that you are able to have nested grids of the children element. So in my demo there is a display: grid on the parent catalogue section and also on both children div elements. This gives fine grain control on how each of the divs are placed and also where each div inner elements are placed.

Fine Grain Control

Let’s specifically look at the catalogue__about-block internal div that has four different content blocks. These blocks are placed on different coordinates on the grid, this was achieved by defining where and how much space each element should take up. 

.catalogue__about-block {
            display: grid;
            grid-template-columns: repeat(6, 1fr);
            grid-template-rows: repeat(5, 1fr);
            grid-gap: 20px;
            height: 900px;
            }

So, now that I’ve defined my grid columns, firstly by using the fraction unit (fr) method, so my grid should have 6 equal columns. Normally you don't have to define the number of rows when using CSS Grid, CSS Grid create rows as need for the content and by being implicit. If you place something outside of the define grid coordinates or due to the amount of content, more grid tracks are needed CSS Grid will automatically create more tracks to fit your content and it will be auto-sized by default. CSS Grids would automatically defined the amount of rows that your grid will take up by default and would let elements fill up the available space. But as I wanted more fine grain control I have defined the amount of rows and and also the height of my grid.
